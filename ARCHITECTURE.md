# Архитектура — «Центр жизни»

Персональный семейный life-dashboard. Самохостинг, mobile-first, русский UI, один общий пароль на вход. Документ описывает, **как устроено приложение**: стек, структура, потоки данных, ключевые паттерны и подводные камни.

---

## 1. Принципы

- **Mobile-first.** Всё проектируется под телефон (узкая колонка, крупные тач-цели), затем масштабируется.
- **Самохостинг на одном устройстве.** Сервер запускается на Mac по необходимости → никаких фоновых демонов; всё считается «лениво» при заходе. Напоминания — баннерами на дашборде (web-push отложен).
- **Приватность по умолчанию.** Все данные чувствительные. Файлы и маршруты — только за авторизацией. Секреты — в `.env`, не в базе.
- **Переиспользование.** Общая инфраструктура (вложения, формы, подтверждения) выносится в `components/` и `lib/` и применяется во всех модулях.

## 2. Технологический стек

| Слой | Технология |
|---|---|
| Фреймворк | **Next.js 16** (App Router, Turbopack), React 19, TypeScript |
| Стили | **Tailwind CSS v4** |
| БД | **SQLite** через **Prisma 6** (`prisma/dev.db`) |
| Авторизация | bcrypt-хеш пароля в `.env` + сессия в подписанной httpOnly-cookie (`jose`) |
| Файлы | Локальная ФС (`var/uploads`) за абстракцией `lib/storage.ts` (легко на S3) |
| Изображения | `sharp` — превью/метаданные |
| Дата/время | `date-fns`; для повторений (план) — `rrule` + `date-fns-tz` |
| Среда | **Node 24** (локально в `~/.local/node-v24`, в системе не было) |

## 3. Структура проекта

```
src/
  proxy.ts                  # middleware (Next 16): защита ВСЕХ маршрутов сессией
  app/
    layout.tsx              # корневой layout (шрифты, globals)
    globals.css
    login/                  # страница входа по паролю
    api/
      files/[id]/route.ts   # защищённая отдача вложений (+ ?thumb, ?download)
    (app)/                  # ГРУППА маршрутов, требующих сессии
      layout.tsx            # оболочка: шапка, навигация, поиск
      page.tsx              # дашборд (карточки разделов)
      medicine/             # ✅ Аптечка
      documents/            # ✅ Документы (+ category/, manage в др. модулях)
      checklists/           # ✅ Чек-листы (+ manage/ редактор)
      schedule/ children/ places/ shopping/ search/   # ⬜ заглушки (Stub)
  components/               # переиспользуемые: attachments, confirm-submit,
                            #   new-category-button, stub, nav и т.д.
  lib/                      # prisma, auth, storage, attachment-*, 
                            #   medicine(+import), routine, document-thumbs
prisma/
  schema.prisma            # модель данных всех 9 модулей
  migrations/              # история миграций
scripts/                   # seed/импорт/сервис (см. ниже)
public/                    # PWA: manifest, icons, sw.js
var/uploads/               # загруженные файлы (НЕ в git)
```

## 4. Маршрутизация и рендеринг

- **App Router.** Серверные компоненты по умолчанию (грузят данные через Prisma), клиентские (`"use client"`) — только для интерактива (формы, оптимистичные галочки, фильтры).
- **Группа `(app)`** объединяет все приватные страницы под общий layout (навигация). `login` и `api/files` — вне группы.
- **Server Actions** — основной способ мутаций (создать/изменить/удалить/отметить). Формы вызывают actions напрямую; после — `revalidatePath`.

## 5. Авторизация и безопасность

- Пароль хранится как **bcrypt-хеш** в `APP_PASSWORD_HASH` (`.env`). Смена — `scripts/set-password.mjs`.
- При входе создаётся **подписанная httpOnly-cookie** (`jose`, секрет `SESSION_SECRET`, срок `SESSION_TTL_DAYS`).
- **`src/proxy.ts`** (middleware Next 16) проверяет сессию на **всех** маршрутах, кроме `login` и статики; без сессии → редирект на `/login`.
- **Файлы — только за авторизацией.** Сканы/фото не лежат в `public`. Отдаёт их `app/api/files/[id]/route.ts`: проверяет сессию (+ proxy), читает из хранилища, ставит правильный `Content-Type`. Параметры `?thumb=1` (превью), `?download=1` (скачать).
- В git **не попадают**: `.env`, `prisma/dev.db`, `var/uploads/` (см. `.gitignore`).

## 6. Слой данных (Prisma + SQLite)

Все модели — в `prisma/schema.prisma`. Ключевые:

- **Person** — люди (kind: SELF/CHILD/FAMILY, цвет для расписания).
- **Document** + **Attachment** — документы и вложения. `Attachment` **полиморфна**: пара `(entityType, entityId)` привязывает файл к любой сущности (`document`, `place`, `categoryCover`, `child`, …) — одна таблица на все модули.
- **Medicine** — аптечка (category, needToBuy, expiryDate).
- **ShoppingList / ShoppingItem** — три списка покупок (kind HOME/SELF/KIDS).
- **RoutineTask / RoutineCheck** — недельная рутина (чек-листы): задача = категория + время дня + дни недели; отметка = факт выполнения на дату.
- **ScheduleEvent / EventException** — расписание (повторения RRULE + переносы/отмены) — *в разработке*.
- **GrowthMeasurement, ChildEvent** — дети (рост/вес, события) — *в разработке*.
- **Place** — места (статус, координаты/адрес) — *в разработке*.
- **PushSubscription, AppSetting** — push-подписки и настройки.

> Особенность SQLite+Prisma: «перечисления» хранятся строками (enum не поддержан), даты рутины — строкой `YYYY-MM-DD` (локальная дата по МСК, чтобы не было сдвига часовых поясов).

## 7. Файлы и вложения (переиспользуемая инфраструктура)

- `lib/storage.ts` — абстракция хранилища (сейчас локальная ФС `var/uploads`, ключи = `<uuid>.<ext>`, превью `<uuid>.thumb.jpg`).
- `lib/attachment-save.ts` — сохранение одного файла (валидация типа/размера, `sharp`-превью).
- `lib/attachment-actions.ts` — server actions: загрузка, удаление, удаление всех для сущности, обложка категории.
- `components/attachments.tsx` — UI: добавить (камера/галерея/PDF), сетка превью, полноэкранный просмотр (лайтбокс), скачать, удалить. Принимает `entityType`/`entityId` — встраивается в любой модуль.

## 8. Ключевые паттерны

- **Оптимистичные галочки** (чек-листы): локальный `useState` через **функциональный** updater + `useTransition` → мгновенный отклик; сервер «флипает» по БД (идемпотентно).
- **Состояние в URL.** Выбранный день чек-листа (`?d=`), активная вкладка/категория — в query-параметрах, чтобы сохранялись при переходах и «назад».
- **Полиморфные вложения** через `(entityType, entityId)` — вместо отдельной таблицы файлов на каждый модуль.
- **Ленивые «проходы».** Никаких фоновых задач: сегодняшние отметки/состояния создаются при заходе (подходит для сервера, который не всегда включён).
- **Массовый импорт** скриптами (аптечка из текста, документы из папки) — отдельно от UI, переиспользует ту же логику хранилища.

## 9. PWA

`public/manifest.webmanifest` + иконки + `sw.js`. Приложение ставится на экран «Домой» на iPhone и выглядит нативно. Полноценный web-push отложен (нужен всегда-включённый хост + HTTPS).

## 10. Конфигурация и подводные камни (Next 16 / Prisma 6)

- **`next.config.ts`:** для загрузки фото/сканов подняты **ДВА** лимита тела — `experimental.serverActions.bodySizeLimit: "100mb"` и `experimental.proxyClientMaxBodySize: 100MB` (по умолчанию 1 МБ и 10 МБ; второй срабатывает, т.к. запрос идёт через `proxy.ts`).
- **Prisma 6:** `where:{ field:{ not:null } }` в `findMany` НЕ работает (фильтровать null в JS); явный `field:null` в `create` отвергается, если поле required.
- **Миграции:** `prisma migrate dev` блокируется в неинтерактивной среде на деструктивных изменениях → такие писать `migration.sql` вручную + `migrate deploy`. После `prisma generate` — **перезапуск dev-сервера**.
- **Гидратация:** значения из `window`/URL читать в `useEffect`, не в инициализаторе `useState`.

## 11. Скрипты (`scripts/`)

| Скрипт | Назначение |
|---|---|
| `set-password.mjs "пароль"` | сменить пароль входа |
| `seed.mjs` | люди + 3 списка покупок |
| `seed-routine.mjs` | задачи недельной рутины (чек-листы) |
| `import-documents.mjs "папка"` | массовый импорт сканов (файл → документ) |
| `set-category-covers.mjs "папка"` | обложки категорий из фото (имя файла = категория) |

## 12. Запуск

```bash
export PATH="$HOME/.local/node-v24/bin:$PATH"
npm install
npx prisma migrate deploy && npx prisma generate
npm run dev    # http://localhost:3000
```
Переменные окружения — см. `.env.example` и раздел 5. Вход — общий пароль (по умолчанию временный, сменить через `scripts/set-password.mjs`).
