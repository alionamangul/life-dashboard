## В очереди ⬜ (рекомендованный порядок)

> **Покупки → Места → Расписание → Дети → Дашборд-обзор → Поиск → Экспорт/деплой.**
> Логика: сперва простое и независимое; движок повторений (Расписание) — до Дашборда (переиспользуется в виджете «ближайшее»); Поиск и Дашборд — последними, когда разделам есть что показать.

### 6. Покупки `shopping` · низкая
Три списка (Дом/Себе/Детям) с табами на одном экране. Списки и люди уже засеяны, миграции не нужны.
- Server actions: добавить пункт, «куплено», удалить, «очистить купленные» (без перезагрузки).
- Клиент: табы (активная — в URL), быстрый ввод, крупные тач-чекбоксы, купленные уходят вниз серыми.
- «Детям» — селект «Кому» + группировка по ребёнку; остальные — по категории.
- **Решить:** разбивка по детям/категориям; пустые или с примерами; баннер «не куплено» на дашборд.

### 7. Места `places` · низкая–средняя
Карточки «хочу посетить / посещено» + фото + кнопка карты. Модель `Place` готова, фото — через вложения (`entityType="place"`).
- Хелперы ссылок: **web Яндекс.Карты (основная)** + deep-link Навигатора (вторичная). Внимание: порядок координат (web ждёт lon,lat).
- Лента с фильтрами (Все/Хочу/Посещено), поиск, быстрое переключение статуса; детальная страница (фото/заметки/ссылка/адрес/кнопки карты/удаление с очисткой вложений).
- **Решить:** одна кнопка карты или обе; маршрут или точка; нужен ли парсер ссылки Я.Карт → координаты; стартовые категории.

### 8. Расписание `schedule` · высокая
Недельная лента тренировок (мои и детей) с повторениями и переносом/отменой отдельной даты. Модели `ScheduleEvent`/`EventException` готовы.
- Библиотеки: `rrule` + `date-fns-tz`.
- **Движок `lib/schedule.ts`:** разворачивает правило повторения в даты видимой недели, применяет отмены/переносы. **Переиспользуется в дашборде.**
- Человеко-понятный конструктор повторения (не сырая строка) с превью на русском; недельная сетка; фильтр-чипы по людям (цвета); меню над итерацией «отменить эту / перенести только эту / изменить всю серию».
- **Риски:** часовой пояс (в БД UTC, «вторник 09:00» — мск); ограничивать бесконечные серии видимым диапазоном; стабильный матчинг переносов по исходной дате.
- **Нужно:** у кого тренировки; реальные примеры расписаний для проверки.

### 9. Дети `children` · средняя
Профили детей с фото, график рост/вес, события (врачи/проверки/поездки) с напоминанием-баннером. Модели готовы.
- Хелперы возраста; CRUD ребёнка/замеров/событий; график — **свой лёгкий SVG** (переключатель Рост/Вес — разные масштабы); напоминания на дашборде.
- **Решить:** нормы ВОЗ на графике (заметно больше работы) или личная динамика; за сколько дней напоминать (сейчас 2).
- **Зависит от:** полного состава семьи.

### 10. Дашборд-обзор + уведомления `dashboard` · средняя
Сводка над карточками: ближайшие события (расписание + дети), просроченные/заканчивающиеся лекарства, незавершённые чек-листы. Баннеры — основной механизм; web-push best-effort (VAPID) отложен.
- Агрегатор `lib/dashboard.ts` (использует движок расписания); виджеты mobile-first; быстрые действия из дашборда.
- **Решить:** окно «ближайших» (предлагаю 7 дней / 5 пунктов); порог «скоро истекает».

### 11. Поиск `search` · средняя
Одно поле → по всем разделам (документы, места, аптечка, покупки, события, люди), результаты сгруппированы со ссылками.
- Серверная функция по источникам + route-handler за авторизацией; живой поиск с задержкой ~280 мс; чипы-фильтры.
- **Важно:** кириллица в SQLite регистрозависима → приводить регистр (иначе «Паспорт» не найдётся по «паспорт»).
- **Решить:** глубина (названия / + заметки); сколько на раздел; история запросов.

### 12. Экспорт/бэкап + документация деплоя `backup` · средняя
Экспорт всех данных: JSON + zip папки `var/uploads` (встроенный zlib). Восстановление CLI-скриптом.
- README: запуск, переменные, смена пароля, бэкап, установка PWA на iPhone (HTTPS!).
- **Важно:** пароль/секреты в `.env`, не в БД — `.zip` не самодостаточен для переезда (написать в инструкции).

## Баги и правки 🐞

- **Обложка категории в Документах не центрируется по лицу.** Сейчас превью кадрируется по центру (`object-cover`), из-за чего лицо человека часто обрезается/уходит вбок. Нужна **центровка по лицу**: либо определение лица и кадрирование по нему (face-aware crop), либо ручная точка фокуса (drag-выбор центра обложки), либо хотя бы смещение кадра к верхней трети, где обычно лицо. Затрагивает плашки на `/documents` и контрол обложки.

## Техдолг и идеи на потом

- **Старые модели чек-листов** (`ChecklistTemplate/Run/...`) в схеме не используются (заменены на `RoutineTask/RoutineCheck`) — удалить отдельной миграцией.
- **Push-уведомления** — теперь есть всегда-включённый прод (**life.alena.com.ru**, HTTPS), можно включать (VAPID-ключи + расписание рассылки просрочки лекарств/событий).
- **Обложки/категории документов** — необязательная мини-миграция для эмодзи/цвета.
- **Глобальный экспорт** и **смена пароля из UI** (сейчас скриптом).
- **Перенести проект из `Downloads`** в незащищённую папку (macOS TCC) — на будущее, чтобы не упираться в права доступа.
- **Репозиторий сделать приватным** (содержит личный план здоровья в `seed-routine.mjs`).
